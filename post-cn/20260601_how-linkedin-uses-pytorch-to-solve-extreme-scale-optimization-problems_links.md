# LinkedIn 如何使用 PyTorch 解决超大规模优化问题

作者：Aida Rahmattalabi、Sanjana Garg、Gregory Dexter、Zhipeng Wang、Ruby Tu、Yuan Gao、Yi Zhang

2026 年 6 月 1 日

## 摘要

本案例研究展示了 LinkedIn 如何对其分布式线性规划求解器 DuaLip 进行重新架构——通过开发 GPU 加速的 PyTorch 版本来应对网络应用等超大规模优化挑战。从 CPU 受限的技术栈完成这一转型后，实现了数量级的加速和高效的多 GPU 扩展，同时降低了工程开销。

## 引言

现代互联网平台不仅仅做预测，它们还要做决策。在 LinkedIn 这样的公司，这些决策驱动着大规模网络应用的智能行为。

在幕后，这些系统中的许多都可以归结为一个看似简单的问题：

*在约束条件下，从数百万（甚至数十亿）个选项中，最优的一组行动是什么？*

这正是线性规划（LP）发挥作用的地方——它是在约束条件下对目标函数进行优化的基础数学框架。在 LinkedIn 的规模下，这些线性规划可能涉及**数亿用户**和**数万亿决策变量**，以及稀疏但高度结构化的约束矩阵。传统的线性规划求解器（如单纯形法和内点法）历来是优化领域的主力。然而，它们依赖于矩阵分解或基更新，在极端规模下，这些操作的内存和计算开销变得难以承受。因此，它们往往无法高效处理现代网络规模的问题。

## 业务挑战

**我们的目标是在相互竞争的目标下优化大规模决策系统。**

示例包括：

- 将职位与潜在求职者进行匹配
- 在排名或推荐系统中平衡多个业务指标
- 优化发送给用户的电子邮件数量

这些本质上是**具有挑战性的优化**问题，改善一个指标（如点击率）可能会损害另一个指标（如投诉率）。在形式上，这些问题被表示为**线性规划**：

- 目标：最大化业务价值（如参与度、收入）
- 约束：执行限制（如预算、公平性、频率）

**关键瓶颈在于可扩展性：随着问题规模的增长，在生产环境中支持快速、可重复的优化，需要在保持稳定性和解决方案质量的同时，实现内存和时间效率的双重保证。**

近年来，一阶方法作为求解此类大规模线性规划的实用替代方案崭露头角。与经典方法不同，这些方法仅依赖梯度信息，避免了昂贵的矩阵分解，使其核心操作以矩阵-向量乘法为主。其中，原始-对偶公式被证明尤为有效：它将线性规划重新表述为鞍点问题，并迭代更新原始变量和对偶变量直至收敛，通常能为生产系统提供足够精确的解。

这一系列工作催生了新一代大规模求解器，包括谷歌的 PDLP 和 LinkedIn 的 DuaLip 等系统。DuaLip 是一个基于岭正则化对偶上升和一阶优化的分布式求解器。它利用匹配问题的可分解结构，并使用加速梯度更新以及高效的投影算子来扩展到极端问题规模。

虽然 DuaLip 证明了一阶方法可以在生产环境中处理网络规模的线性规划，但其基于 Scala/Spark 技术栈的原始实现从根本上仍然受 CPU 限制。这限制了它充分利用现代硬件加速器的能力。此外，其与模式绑定的模板驱动接口难以扩展到新的问题公式，减慢了不断演变的使用场景的迭代速度。

受这些局限性的启发，我们在 **PyTorch 和 GPU 加速**的基础上重新架构了 DuaLip 求解器栈，形成了 DuaLip-GPU——一个用于工业规模优化的现代、灵活且可扩展的系统。

## LinkedIn 如何使用 PyTorch

为了应对这些挑战，我们提出了 **DuaLip-PyTorch** 作为大规模优化的核心执行引擎——而不仅仅是深度学习。该系统围绕算子级的数组/张量编程模型（遵循 PyTorch 的动态图定义范式）构建，而非任务级的"调用求解器"API。

具体而言，热路径被表示为稀疏矩阵-向量操作和分块投影上的显式数据流，由轻量级最大化器编排。这一设计边界是有意为之的：它暴露了主导运行时的内核，支持灵活选择稀疏布局和投影算子，并自然地映射到 GPU 执行——所有这些都无需更改核心优化循环。

## 用 PyTorch 解决 AI 挑战

PyTorch 提供原生 GPU 加速、针对稀疏和密集计算的灵活张量抽象，以及用于梯度计算的高效矩阵-向量操作。这些能力结合在一起，使得大规模线性规划求解在结构上类似于神经网络训练，但使用了优化特有的原语。在 LinkedIn，这些特性帮助解决了三大系统和优化挑战。

首先，针对包含数十亿到数万亿变量的超大规模线性规划，使用**稀疏张量操作**和**批量投影内核**实现，使 GPU 上的高效执行成为可能。

其次，通过将变量分区到各 GPU，同时通过 all-reduce 和 broadcast 等集合通信模式复制和同步对偶变量，实现分布式优化，从而允许跨设备的近线性扩展。

第三，通过结合行归一化和缩放（以改善条件数）、正则化延续策略，以及包括 AGD 和 FISTA 变体在内的可扩展一阶优化方法，提高收敛速度。这些改进在保持精度的同时显著减少了求解时间。

## 使用 PyTorch 的收益

使用 PyTorch 使 LinkedIn 能够：

- **实现数量级的加速**，优于基于 CPU 的系统
- 从单 GPU 到多 GPU 系统的高效扩展
- 支持**灵活、可扩展的线性规划公式**
- 减少新优化问题的工程开销
- 将机器学习与优化整合到统一的技术栈中

最重要的是，通过围绕 GPU 高效稀疏线性代数重构求解器，它实现了**在此前不可行的规模下进行生产级优化**。

DuaLip-Pytorch 中的主要计算由重复的稀疏矩阵-向量乘法和投影更新组成，这自然地映射到高吞吐量的 GPU 执行。通过将这些操作表达为 PyTorch 中的批量张量内核，并通过同步集合通信将其分布在多个 GPU 上，该系统与原始基于 CPU 的实现相比，实现了显著更低的每迭代求解时间。
